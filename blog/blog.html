Analogy to describe HTML, CSS and the DOM

HTML contains the content that will be displayed on a webpage. CSS contains the style rules that will be applied to that content. HTML and CSS are kept in separate documents to make a wesbite more modular and easier to scale.

The DOM is the combined HTML and CSS that the browser uses to display content on a webpage. It could be exactly the same as the HTML code, but it could also differ due to Javascript or other code which changes the content before or after it's rendered.

Now I need an analogy. How about an excel spreadsheet which includes macros that update certain inputs. If you send that spreadsheet (the HTML code in this analogy) old mate Trevor, he could open that using his Microsoft sofware (the browser) and see exactly the same "HTML content". However the macros could pull in new data to create a modified version of the spreadsheet that was originally sent to Trevor. This modified version (the "DOM") is what is rendered through Trevor's "browser". It's analagous to the DOM that your Chrome brower renders after compiling the HTML, CSS, and any other scripts included in the code.

What is meant by boxifying design

Every webpage consists of an underlying grid. Every bit of content will sit somewhere on this grid.

In order to modify the default layout, we need to define the position and charactistics of each bit of content. The easiest way to do this is to group relevant chunks of content into boxes (also called containers). Once we have content in a container, we can apply styling and rules to that container to manipulate its characteristics. These design rules can then be easily applied to other relevant containers by giving them the same "class" or "id".

Boxifying design makes code easier to understand, easier to manipulate, more modular, and more scalable.

What is the box model

As above, the box model refers to the fact that the content on every webpage is attached to an underlying grid. An understanding of the box model is required in order to manipulate the design of the content.

What is grid based design

Each webpage is attached to an underlying grid. This grid allows us to position content.

What is all the hype about responsive webpages

People browse the web on different devices, which have a wide range of screen sizes and functionality. Responsive webpages allow us to design a single webpage that will provide a good experience for users across a range of devices.

What is semantic structure

Semantic structure is when you use specific HTML tags to help the browser (and anyone reading your code) to better understand content.

Semantic tags have meaning, as opposed to generic tags like <div>. Examples of semantic tags include <article> and <form>. These semantic tags clearly define the content for the browser and developer.
