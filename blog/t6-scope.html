"Scope", and how it works in JavaScript

We use HTML to create elements on a webpage. Then we use other languages, like CSS and JavaScript, to write rules that modify those elements. Sometimes we want these rules to apply to all elements. Sometimes we want these rules to only apply to a specific element. We do this by giving the element a class or id.

JavaScript has an analagous concept for defining how broadly a piece of JavaScript code applies within a JavaScript file. This is referred to as "scope".

"Global scope" is when a piece of code applies to the whole file. "Local scope" is when a piece of code is restricted to a limited part of the file. As an example, you may want to create a variable within a function that should not be accessed outside of the function. So you'd create the variable within the function body to give it local scope. On the contrary you may want a variable to be accessible by multiple functions, so you'd create the variable outside of any function to give it global scope.
